Para la parte de la carga de datos
se desarrollo un sistema web,

hecho en PHP sobre Laravel,

a través del sistema se cargaron los datos de los arcos
y etiquetas de polígonos de manzanas,

Los listados de viviendas del precenso (ADRA)

Y una tabla de relación entre radios y localidades, departamentos, aglomerados con la estructura formal.

A medida esos datos se cargan en el servidor web
se procesaron los datos para que entren 
en la base de datos.

Se completan campos, de trata de ordenar las tablas,
se hacen arreglos en algunos campos y generar validaciones
en cuanto a los listados de viviendas y su correspondencia 
con la cartografía, para luego pasar a 
la segmentación desde el sistema web.

---
Los elemento disponibles a partir de los cuales
se pensó la carga de datos fueron:
 Conteos: _Cantidad de viviendas_ por lados de manzana
 o Listado de direcciones según tipo de vivienda donde estaba el orden del recorrido dentro del lado de la manzana.
 
 
 ---
 
Para todo el desarrollo se utilizó software libre
 sobre linux,
 las bases de datos Postgresql con Postgis
 y postgis_topology, dónde se cargaron los datos
 espaciales y los tabulares.
 
El sistema web se desarrollo con Laravel
 en su mayor parte con varios plugins de Laravel
 
Utilizamos 3 servidores, uno de Desarrollo,
 uno de Testing y uno de Producción,
 todos con GNU/Linux CentOS, Apache

En la parte web se utilizó Html5, css3, boostrap,
svg, jquery, webpack y otras herrameintas disponbiles.

Se utilizó mucho GDAL, ya que muchos de los archivos que
teníamos eran E00 que son de exportación de ARC/Info
y shapes, éstpo depende de cada provincia, algunas trabajan con e00 
y otras trabjan con shapes.

El código desarrollado, lo dividimos en 3 partes, 
- una parte del Core principalmente, 
- una parte web 
- una parte correspondiente al PLUGIN de QGis.

Todos los desarrollos se manejaron 
un repositrio de Git (en gitlab institucional y GitHub)

Se utilizó OpenStreetMap como capa base 
para todas las pruebas que hicimos.

Útilizamos software recomendado por OSGeo, entre ellos OSGeoShell, PostGis, Gdal, OGR, QGis.

La parte del sistema del CORE, algoritmos de intercambio, mutaciones de segmentaciones y cáculos,
se hicieron mayormente con Python, 
GeoJson se utilizó para el intercambio de datos en los sistemas, 
GNU/Linux para todo tipo de imagenes y demas. 
QGis principalmente para el 
plugin final de Salida Gráfica. 

Después de la carga de datos de localidades y de aglomerados,
se puede navegar por el sistema web, acceder a una localidad 
e ir a segmentar un radio de una determinada localidad.
Con éste formulario que se ve, se permite segmentar un radio,
previsualiza la cantidad de viviendas que se han encontrado para ése radio y
permite seleccionar los 3 métodos de segmntación:
- Manzanas independientes
- Lados completos
- Combinados
que se usa para los radios mixtos esos mezclados en donde las viviendas 
no estan distribuídas y no están agrupadas, y se setean los parámetros de cantidad de 
viviendas deseadas, una cantidad maxima de viviendas deseadas, una  mínima 
y un parámetro mas que es Manzanas Indivisibles que lo que hace es mantener 
las manzanas unidas cuando tienen menos de cierta cantidad de viviendas
Son parámetros con los cuales el operador puede jugar e ir segmentando
un radio hasta encontrar una segmentación que le parezca adecuada.

Cada vez que se corren estos algoritmos de segmentación se producen un 
momtón de procesos además de generar la segmentación se hace una versión textual,
resumen textual de la segmentación que es una R3 donde para cada segmento
se describe de manera compacta y legible por humanos el segmento generado
con las cantidad de manzanas o lados o pedacitos de lados que haya que recorrer.

En todos estos procesos lo que se termina generando en un esquema 
que nosotros definimos para localidades, un aglomerado puede tener 
muchas localidades como tenemos definido nosotros tenemos provincia, 
departamento y un codigo de localidad, dividimos esa estructura
en esquemas dentro de Postgres y generamos un momtón de tablas, 
la información que se ha cargado serían los arcos, la tabla arc la tabla lav que 
son las etiquetas y una tabla listado con las viviendas de la C1
Todas las otras tablas son productos de procesos, que se fueron generando 
se carga los arcos en la topología, se usan las etiquetas para generar las 
manzanas los radios y las fracciones como sumatoria de manzanas se generan 
los radios, como sumatoria de radios se generan las fracciones.
Se hace una georeferenciación de ése listado con los arcos, 
se genera una tabla del listado ya georeferenciado para utilizar, 
se genera una tabla de segmentación y una tabla de R3 con éste resumen 
que tenemos.
